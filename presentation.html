OOP Concept: Inheritance

include/person.hpp

class Person {
protected:
    std::string name_;
public:
    virtual std::string getDescription() const = 0;
    // ^ pure virtual — abstract base
};

class Student : public Person {
    std::string course_, grade_;
public:
    double gradeToPoints() const;  // A → 4.0, B+ → 3.3, etc.
    std::string getDescription() const override;
};

Key Points

  • Person is an abstract base class — cannot be instantiated directly
  • Student inherits name_ and adds course/grade
  • Virtual destructor ensures proper cleanup through base pointers
  • getDescription() is overridden to return a formatted string
  • Demonstrates: is-a relationship, code reuse, interface enforcement

OOP Concept: Polymorphism

include/certificate.hpp

class Certificate {                    // Abstract base
virtual std::string generateHTML() const;
virtual std::string getTypeName() const = 0;
};

class CertificateOfExcellence : public Certificate {
    std::string getTypeName() const override { return "Excellence"; }
};

Design Pattern: Strategy

CertificateTemplate defines an interface; concrete subclasses provide different visual styles. The Certificate class delegates all HTML rendering to the template.

class CertificateTemplate {              // Strategy interface
virtual std::string header(...) const = 0;
virtual std::string styles() const = 0;
virtual std::string body(...) const = 0;
};

// Three concrete strategies:
class FormalTemplate : public CertificateTemplate { ... };
class ModernTemplate : public CertificateTemplate { ... };
class MinimalistTemplate : public CertificateTemplate { ... };

Why Strategy?

  • New templates can be added without changing Certificate
  • Template can be swapped at runtime via a dropdown
  • Each template is independently testable
  • Open/Closed Principle — open for extension, closed for modification

Design Pattern: Template Method

The generateHTML() method in the base Certificate class defines the skeleton algorithm. Subclasses override getTypeName() to plug in at the right step.

std::string Certificate::generateHTML() const {
    // Fixed algorithm skeleton:
    std::string typeName = getTypeName(); // ← virtual
    std::string html;
    html += template_.header(typeName);  // ← strategy
    html += template_.styles();          // ← strategy
    html += template_.body(..., typeName); // ← strategy
    html += "</body></html>";
    return html;
}

How It Works Together

Template Method (Certificate) + Strategy (CertificateTemplate)

1. Get certificate type nameSubclass (virtual)
2. Write <head> + <title>Template (strategy)
3. Write CSS stylesTemplate (strategy)
4. Write body with student dataTemplate (strategy)
5. Close HTML tagsBase class (fixed)

Design Pattern: Factory

Two factory functions encapsulate object creation — the caller specifies an enum and gets back the correct polymorphic type via std::unique_ptr.

makeTemplate()

std::unique_ptr<CertificateTemplate>
makeTemplate(TemplateStyle style) {
    switch (style) {
        case TemplateStyle::Formal:
            return std::make_unique<FormalTemplate>();
        case TemplateStyle::Modern:
            return std::make_unique<ModernTemplate>();
        case TemplateStyle::Minimalist:
            return std::make_unique<MinimalistTemplate>();
    }
}

makeCertificate()

std::unique_ptr<Certificate>
makeCertificate(CertType type,
    const Student& s,
    const CertificateTemplate& t) {
    switch (type) {
        case CertType::Excellence:
            return std::make_unique<CertificateOfExcellence>(s, t);
        // ... Completion, Participation ...
    }
}

Usage in generator.cpp:

// The generator doesn't need to know the concrete types:
auto tmpl = makeTemplate(style);   // returns FormalTemplate, ModernTemplate, or MinimalistTemplate
auto cert = makeCertificate(type, student, *tmpl); // returns the right Certificate subclass
file << cert->generateHTML(); // polymorphic call — correct behavior resolved at runtime

Sorting (std::sort with lambdas)

void sortByGrade(bool descending) {
    auto vec = getAllStudents();
    std::sort(vec.begin(), vec.end(),
        [descending](const Student& a,
                     const Student& b) {
            return descending
                ? a.gradeToPoints() > b.gradeToPoints()
                : a.gradeToPoints() < b.gradeToPoints();
        });
}

Case-Insensitive Search

Student* findStudent(const std::string& name) {
    std::string lower = toLower(name);
    auto it = std::find_if(students_.begin(),
        students_.end(), [&](auto& pair) {
            return toLower(pair.first) == lower;
        });
    return it != students_.end() ? &it->second : nullptr;
}
